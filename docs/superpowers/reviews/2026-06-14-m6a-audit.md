## ✅ Corrigé — code (commit `5f00403`)
- **[CRITIQUE] Timeout HTTP** sur tous les appels Discogs/pochette (`ureq` n'a pas de read-timeout
  par défaut → un serveur qui pend gelait le thread IPC, donc l'UI, indéfiniment). 15 s (search +
  tracklist), 20 s (pochette).
- **[Important] `apply_identity_cmd` gardé** à un track connu avant tout travail (téléchargement /
  écriture DB) — comme `identify` l'est via `reconcile_track`.
- **[Important] Pochette plus périmée au ré-identify** : `cover_path` écrit inconditionnellement
  (avant, `COALESCE` gardait la pochette de l'ancienne release si le nouveau download échouait).
- **Maintenabilité** : helper `map_ureq_err` partagé (dédup), log `warn` quand un appel tracklist
  est rate-limited (diagnostic), retrait des `#![allow(dead_code)]` sur les modules M6a câblés
  (`metadata/*`, `genres`) → les vrais warnings de code mort ressortent désormais.
- **Tests** : `clean_artist` (artefacts `*` / `(2)`, parenthèses réelles gardées, multi-artistes).

## ⏳ Différé (raison)
**UX (à faire avec toi / plus gros)**
- Rate-limit : compte à rebours + ré-essai auto (au lieu du message seul). [I2]
- État « action en cours » sur tout le footer, pas seulement le bouton Ranger. [I5]
- Message dédié « Token invalide » (401) au lieu de « injoignable ». [I6]
- Indicateur « enregistrement… » pendant le debounce du token (600 ms). [m4]
- « Identifier + Ranger » en un geste ; aperçu du nom au survol d'un candidat ; état d'erreur
  waveform visible. [N1–N4]

**Code**
- Test `apply_identity(track_id inexistant)` : nécessite d'activer `PRAGMA foreign_keys=ON` dans le
  harness de test (off en in-memory) — sinon l'orphelin s'insère en test alors qu'en prod le FK le
  rejette. La garde IPC [I2] couvre le risque réel ; test différé.
- Pochette PNG sauvée en `.jpg` (MIME forcé JPEG) — cosmétique, lofty l'embarque quand même. [m4]
- AcoustID = 2ᵉ provider par le son (réutilise nos empreintes) → cf. note de cadrage `la suite`.

## Working-as-intended (ne pas changer)
- Échec d'un appel tracklist individuel avalé (best-effort) — un détail KO ne doit pas tuer
  l'identification.
- Version (remix/dub) conservée du nom local — Discogs n'expose pas la version par piste.
- `Confidence::Green` après un match Discogs — c'est un renommage fiable confirmé.
- Genres : structurés/ordonnés en DB (`track_genres`), joints `A; B` dans le tag fichier
  (le multi-item ne round-trip pas sur ID3 ; Rekordbox lit un champ unique).
